# DawnNew 重建计划（最终版）

## 目标

基于 DawnHub 的优秀部分，搭建一个干净、模块化、可自进化的 DawnNew，完全去除 Claude Code 残留，同时保留核心能力（代码生成、审查、记忆、自进化、humanBrowser）。

## 安全措施（必须先做）

1. 整个过程在 `D:\AI\DawnNew` 文件夹操作，不动 DawnHub。
2. 每阶段开始前执行 `git init + git add . + git commit -m "阶段X"`（方便回滚）。
3. 重要文件复制前先备份。

---

## 分阶段重建计划

### 阶段 0：初始化（10 分钟）

- 创建干净的 DawnNew 项目结构（严格按给定架构）
- 初始化 git
- 复制 `package.json` 和核心配置文件

### 阶段 1：核心架构搭建 + 清理 Claude 残留（今天完成）

- 创建所有目录（engine、capabilities、memory、evolution、tools、cli、utils、packages）
- 从 DawnHub 只复制核心文件（不是全部复制）
- 彻底清理 Claude 残留（删除目录 + 全局搜索替换品牌文字）
- 创建 `new-main.ts` 作为临时入口

### 阶段 2：引擎 + 记忆 + 自进化（2-3 天）

- 移植并优化 IntentEngine、Orchestrator、AgentCore
- 完善 MemoryManager（三层记忆）
- 接入 SelfEvolutionEngine（任务后自动分析 + 技能提取）

### 阶段 3：能力层移植（1-2 天）

- humanBrowser（真实 fetch + 摘要）
- CodeReviewEngine（严格审查）
- 其他工具（FileOps、WebSearch 等）

### 阶段 4：REPL + DawnPanel 集成（1-2 天）

- 完善交互式 REPL（new-main.ts）
- DawnPanel 基础集成（聊天界面 + 调用新引擎）

### 阶段 5：测试 + 最终优化

- 全面测试 + 修复
- 性能优化 + 文档

## 架构总览

```
DawnNew/
├── Dawn/                     # CLI 核心（Bun + TS）
│   ├── src/
│   │   ├── engine/           # 新核心：IntentEngine + Orchestrator + AgentCore
│   │   ├── capabilities/     # 所有能力（Browser、CodeReview、FileOps、WebSearch...）
│   │   ├── memory/           # 三层记忆系统（Session / Persistent / Skill）
│   │   ├── evolution/        # 自进化引擎（核心竞争力）
│   │   ├── tools/            # 工具系统（统一 Tool Calling 适配器）
│   │   ├── cli/              # 命令行入口
│   │   └── utils/            # 通用工具
│   └── packages/             # 本地私有包 (@dawn/core, @dawn/memory...)
│
├── DawnPanel/                # Tauri 桌面客户端（重点重构）
│   └── src/ (React + 新 IPC)
│
└── tests/                    # 全新测试体系
```

## 核心原则

1. **零 Claude Code 残留**：任何来自 Claude Code 的文件、import 路径、品牌文字都不允许出现
2. **全自构建**：每个文件都经过理解和重写，不是批量复制
3. **模块化**：引擎、能力、记忆、进化各自独立，通过接口通信
4. **可自进化**：SelfEvolutionEngine 是核心竞争力，任务后自动分析 + 技能提取
